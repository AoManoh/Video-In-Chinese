# 🚀 快速测试指南（5分钟）

## 启动Mock模式

```bash
cd client
npm run dev
```

## 测试流程

### 1️⃣ 配置页面（1分钟）

访问：`http://localhost:5173`

**填写任意字符串**（不会真实调用API）：

```
ASR服务商: openai-whisper
ASR API Key: test-asr-key-123

翻译服务商: google-gemini  
翻译 API Key: test-translation-key-456

声音克隆服务商: aliyun-cosyvoice
声音克隆 API Key: test-voice-key-789
```

点击"保存配置" ✅

---

### 2️⃣ 上传页面（1分钟）

点击顶部"上传视频"

**选择任意视频文件**（不会真实上传）：
- 可以是手机录制的视频
- 可以是下载的MP4文件
- 文件大小随意（不会真实传输）

点击"开始上传" ✅

---

### 3️⃣ 任务列表（3分钟）

**自动跳转到任务列表**，观察状态变化：

- `0秒`：排队中（灰色）
- `3秒`：处理中（橙色旋转）
- `10秒`：已完成（绿色对勾）

点击"下载结果" → 浏览器下载一个模拟文件 ✅

---

## 验证Mock模式的方法

### 方法1：查看控制台输出

按 `F12` 打开开发者工具，应该看到：

```
[Mock] Mock数据已启用
[Mock] GET /v1/settings
[Mock] POST /v1/settings
[Mock] POST /v1/tasks/upload
[Mock] 任务abc12345状态变更: PROCESSING
[Mock] 任务abc12345状态变更: COMPLETED
```

### 方法2：查看Network标签页

打开开发者工具 → Network标签页：

- ✅ 没有真实的网络请求
- ✅ 没有发送到 `localhost:8080`
- ✅ 所有请求都是本地模拟

### 方法3：断开网络

断开WiFi或拔掉网线：

- ✅ 应用仍然可以正常使用
- ✅ 所有功能都正常工作
- ✅ 证明没有真实的网络请求

## 常见误解

### ❌ 误解1：需要填写真实的API Key

**正确**：可以填写任意字符串，Mock不会验证

### ❌ 误解2：会调用OpenAI/Google API

**正确**：完全不会，所有请求都被拦截

### ❌ 误解3：会上传真实的文件

**正确**：文件不会真实上传，只模拟上传过程

### ❌ 误解4：需要启动后端服务

**正确**：完全不需要，Mock是纯前端的

## 下一步

Mock测试通过后，等待后端实现，然后：

```bash
# 修改一行配置
VITE_USE_MOCK=false  # .env.development

# 重启
npm run dev
```

**这时才会调用真实的API！**
